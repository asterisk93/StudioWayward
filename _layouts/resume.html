"SW Model Works" Scale Model Showcase website
Studio Wayward Model Works is a website I built to showcase finished pieces from my scale modelling hobby, using Jekyll's blog framework. "Posts", or in this case, individual models, have their raw data stored in markdown files, which are then used in conjunction with my html templates to generate the model pages. The site is styled entirely from scratch using vanilla CSS, which I've linked in such a way so as to exclude any and all Jekyll theme defaults.
❮
1/4
This is the Wayward Model Works homepage. The section on the left contains the 'featured' model preview (set by a flag in the source markdown), which displays a rotating animated gif of the model, along with the model scale and description. The section on the right lists all the remaining, non-featured models, along with thumbnail previews that play as animated gifs on mouseover.
2/4
The dedicated page for each model features an image gallery containing a higher resolution gif of the model on a 360 degree turntable, as well as other photos of the model. A 'dot' bar is included for ease of gallery navigation.
3/4
The model's scale (if applicable) and description are displayed underneath the gallery.
4/4
Each individual model is stored as a single markdown file, using yaml frontmatter that contains links to the high and low resolution animated gifs (hosted externally to save space), the 'assets' subfolder containing the gallery images, as well as paths for each of those images.

"Wayward Photography" Photo Portfolio website
Wayward Photography is another site made under the 'Studio Wayward' banner to serve as a portfolio/gallery site for my photography hobby. Like Model Works, it's a jekyll based blogsite styled from scratch, but with a different markdown file structure that allows for individual photos to be managed independent of their bespoke photosets. I opted for a color scheme with dark greys and blacks, accented with a very rich yellow, so as to better fit the urban aesthetic of my photography.
❮
1/5
This is the Wayward Photography homepage, containing a 5x4 grid of the 20 most recently uploaded photos regardless of associated photoset. A link to continue loading the full photo feed in reverse chronological order is located under the grid.
2/5
Clicking one of the photo thumbnails causes the image to pop up in a modal gallery. If triggered from the homepage, a link to the photoset ("recurring theme") is shown underneath the image, alongside its caption, if applicable. Left and right arrows allow for navigation through all the photos with thumbnails displayed on the page.
3/5
The navigation bar at the top of the page contains a link to load the full gallery in reverse-chronological order, and a drop-down menu listing all of the photosets, also in reverse-chronological order.
4/5
Each photoset page displays the 4 x n photogrid with the photos from that set, underneath the set's title and a short description/preamble.
5/5
Unlike the 1:1 page-to-markdown structure that was used for the Model Works site, Wayward Photography needed a way to treat both the photoset and its individual photo as objects that could be managed with Jekyll. To achieve this, each photoset and each individual photo has its own dedicated markdown file.

"Bite-Size Reviews" Movie Review blogsite
Bite-Size Reviews is my first attempt at making a Jekyll-based blog site styled entirely from scratch with vanilla CSS. It was finished about a month or so before Studio Wayward, but came to fall under the Wayward banner all the same. Up to this point I had been reviewing movies on facebook for a number of years, and when faced with the fact that I had garnered a slight following, I decided to create a place online where I could formally collect all of my reviews for ease of retroactive reading and access going forward.
❮
1/6
The Bite-Size Reviews homepage features the BSR logo prominently over a looping gif of countdown film reel that I recorded at a friend's house (not stock footage). Also prominent are the two large buttons leading to the reviews (left) and the articles (right).
2/6
Below the post category buttons are the post previews of the x most recent reviews/articles in reverse-chronological order. The amount previewed on the homepage changes depending on however many posts I add within a given update (usually 2-4). I try to keep at least one or two posts from the previous update on the homepage alongside the posts from the new update.
3/6
The "Search" button on the far right of the top navigation bar leads to the search page, where readers can search for reviews or articles via title. This is accomplished by iterating over the posts beforehand to build a json structure of post data and div ID, which is then compared to the user query when submitting a search, causing the post previews for any matching posts to be triggered into displaying via the ID.
4/6
Each post, be it considered review or article, has a corresponding markdown file containing data to be plugged into my "post" html template by jekyll in order to generate the individual post pages. The review or aticle body itself makes up the 'content' section of the markdown file, while the yaml frontmatter contains data like the movie title, its review score, a path for its poster/image file, etc.
5/6
This is the end result of the values from the previous markdown file being plugged into my 'post' Jekyll layout to generate the final page displayed on the website.
6/6
I made a fun little parody 404 graphic that would be thematically appropriate for a movie review site should a reader find themselves lost. If all continues to go well, nobody will ever see it.

"Wayward's Son" Content Submission Applet
"Wayward's Son", named after the classic rock song, is a companion applet coded in Python with a basic TKinter interface. It is designed to automate and streamline some of the more tedious steps involved in post management for the Studio Wayward jekyll sites. (markdown generation, folder structures, asset naming, etc.)
❮
1/5
We start off by choosing which site we'll be updating. Current functionality allows for the the creation of a new model, a new photoset, adding new photos to either that or an existing photoset, and the writing of a review or article.
2/5
The screen for model creation asks us for data values and details while taking care of the markdown formatting. Here, we're selecting a source folder for the images that will show up in the model's gallery. The program iterates through the source folder images, renames them in accordance with the Wayward file structure, and moves them to an appropriate assets subfolder.
3/5
Let's write a review. We input the required info (title, rating value, date, and so on), write the body of our review, select the poster image file (for this example, we'll use B.B. King), and hit Submit.
4/5
Here, we can see that our submitted values have been successfully carried over into a properly formatted markdown file with yaml frontmatter. The program gives us the option to terminate or to continue and start the post creation process again from the beginning.
5/5
Finally, after a quick push to the git repo, we see what our test review looks like on Bite-Size Reviews, B.B. King and all, noting that the newline formatting has carried over from the applet, to the markdown file, and finally, to our site post.

"InvestiGate" Educational Logic Gate Game/Sandbox
InvestiGate was an academic group project designed and completed in my final year of university. InvestiGate was built using the PhaserJS game dev framework, and designed to be an interactive tool that would use gamification principles and experimentation to make circuit logic easier to learn. The prototype we submitted consisted of five levels meant to teach players about the function of AND, OR, and NOT gates, as well as an open sandbox mode allowing players to design their own gate/wire structures and observe the interactions that result.
❮
1/6
The InvestiGate main menu.
2/6
An InvestiGate level, showing some Click-and-drag gameplay in progress. Players read the level prompt at the top of the screen, and click and drag gate sprites from the tray on the right, and onto the play area. Players then draw wires from the triangular input nodes assigned by the level, to the nodes on the gate sprite, and then from the gate sprite to the output node on the right.
3/6
The level select screen was made out of nodes that would turn green to indicate the levels the player has unlocked. The 'switch' for the level would close if it was completed successfully. In this case, the player has successfully completed level one, and can choose to move on to level 2 or to replay level 1.
4/6
This is a level abstract/mockup document I made for use in the layout planning and level design phase, Featuring a more cartoonish placeholder logo that we later adapted to be more sleek and modern.
5/6
Wires are color-coded to show which signal (either a 1, or a 0) is running through them.
6/6
Upon completing a level, a success dialog would pop up, either once again outlining the principle that the player had learnt by completing the level, or in this case, congratulating the player for completing all of the levels in a section.
